## Step 2: Import the Database

The `TiendaRopa.sql` file contains the complete database schema and sample data.

<Steps>
  <Step title="Locate the SQL File">
    Find `TiendaRopa.sql` in your project directory:
    
    ```
    CRUD Tienda Ropa/TiendaRopa.sql
    ```
  </Step>
  
  <Step title="Import via phpMyAdmin">
    In phpMyAdmin:
    
    1. Click on the **Import** tab in the top menu
    2. Click **Choose File** and select `TiendaRopa.sql`
    3. Ensure format is set to **SQL**
    4. Scroll down and click **Go/Import**
    
    <Check>You should see a success message: "Import has been successfully finished"</Check>
  </Step>
  
  <Step title="Verify Database Creation">
    Look at the left sidebar. You should now see a database named **tienda_ropa**.
    
    Click on it to expand and verify all 8 tables were created:
    - actualizacion
    - categoria
    - color
    - empleado
    - movimiento_stock
    - prenda
    - proveedor
    - registro
    - talla
  </Step>
</Steps>

<Warning>
  The SQL file automatically creates the database. If a `tienda_ropa` database already exists, you may want to drop it first to avoid conflicts:
  
  ```sql
  DROP DATABASE IF EXISTS tienda_ropa;
  ```
</Warning>

## Step 3: Understand the Database Schema

### Core Tables

<Tabs>
  <Tab title="prenda (Garments)">
    The main inventory table storing all clothing items.
    
    ```sql
    CREATE TABLE `prenda` (
      `id_prenda` int(11) NOT NULL AUTO_INCREMENT,
      `nombre` varchar(100) NOT NULL,
      `precio` decimal(10,2) NOT NULL,
      `stock_actual` int(11) DEFAULT 0,
      `id_categoria` int(11) NOT NULL,
      `id_talla` int(11) NOT NULL,
      `id_color` int(11) NOT NULL,
      PRIMARY KEY (`id_prenda`)
    );
    ```
    
    **Key Relationships**:
    - Links to `categoria` (clothing type)
    - Links to `talla` (size)
    - Links to `color` (color)
  </Tab>
  
  <Tab title="categoria (Categories)">
    Defines clothing categories.
    
    ```sql
    CREATE TABLE `categoria` (
      `id_categoria` int(11) NOT NULL AUTO_INCREMENT,
      `nombre` varchar(50) NOT NULL,
      `descripcion` text DEFAULT NULL,
      PRIMARY KEY (`id_categoria`)
    );
    ```
    
    **Sample Data**:
    1. Caballero - Men's formal and casual wear
    2. Dama - Women's dresses and blouses
    3. Infantil - Children's clothing (2-12 years)
    4. Deportiva - High-performance athletic wear
    5. Accesorios - Belts, scarves, basic jewelry
  </Tab>
  
  <Tab title="movimiento_stock (Stock Movements)">
    Tracks all inventory changes.
    
    ```sql
    CREATE TABLE `movimiento_stock` (
      `id_movimiento` int(11) NOT NULL AUTO_INCREMENT,
      `fecha` datetime DEFAULT current_timestamp(),
      `tipo_movimiento` varchar(10) NOT NULL,
      `cantidad` int(11) NOT NULL,
      `id_prenda` int(11) NOT NULL,
      `id_empleado` int(11) NOT NULL,
      PRIMARY KEY (`id_movimiento`)
    );
    ```
    
    **Movement Types**:
    - `entrada` - Stock received
    - `salida` - Stock sold/removed
    - `ajuste` - Inventory adjustment
  </Tab>
  
  <Tab title="actualizacion (Price Updates)">
    Maintains price change history.
    
    ```sql
    CREATE TABLE `actualizacion` (
      `id_actualizacion` int(11) NOT NULL AUTO_INCREMENT,
      `fecha` datetime DEFAULT current_timestamp(),
      `precio_anterior` decimal(10,2) NOT NULL,
      `precio_nuevo` decimal(10,2) NOT NULL,
      `id_prenda` int(11) NOT NULL,
      `id_empleado` int(11) NOT NULL,
      PRIMARY KEY (`id_actualizacion`)
    );
    ```
    
    Automatically timestamped when prices are updated.
  </Tab>
</Tabs>

## Step 4: Configure Database Connection

The database connection is managed through `db.php` using PDO.

### Default Configuration

```php tienda_ropa/db.php
<?php
$host = "localhost";
$db_name = "tienda_ropa";
$username = "root"; // Usuario por defecto en phpMyAdmin
$password = "";     // Contraseña por defecto vacía en XAMPP

try {
    $conn = new PDO("mysql:host=$host;dbname=$db_name;charset=utf8", $username, $password);
    $conn->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
} catch(PDOException $e) {
    echo "Error de conexión: " . $e->getMessage();
}
?>
```

### Custom Configuration

<Info>
  Only modify `db.php` if you've customized your MySQL installation with:
  - A different root password
  - A custom database user
  - A non-standard port
  - Remote database access
</Info>

If needed, update the connection parameters:

<CodeGroup>
```php Custom Password
<?php
$host = "localhost";
$db_name = "tienda_ropa";
$username = "root";
$password = "your_mysql_password"; // Add your password here

try {
    $conn = new PDO("mysql:host=$host;dbname=$db_name;charset=utf8", $username, $password);
    $conn->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
} catch(PDOException $e) {
    echo "Error de conexión: " . $e->getMessage();
}
?>
```

```php Custom Port
<?php
$host = "localhost";
$port = "3307"; // Custom port
$db_name = "tienda_ropa";
$username = "root";
$password = "";

try {
    $conn = new PDO("mysql:host=$host;port=$port;dbname=$db_name;charset=utf8", $username, $password);
    $conn->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
} catch(PDOException $e) {
    echo "Error de conexión: " . $e->getMessage();
}
?>
```

```php Custom User
<?php
$host = "localhost";
$db_name = "tienda_ropa";
$username = "tienda_user"; // Custom database user
$password = "secure_password";

try {
    $conn = new PDO("mysql:host=$host;dbname=$db_name;charset=utf8", $username, $password);
    $conn->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
} catch(PDOException $e) {
    echo "Error de conexión: " . $e->getMessage();
}
?>
```
</CodeGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Error: Access denied for user 'root'@'localhost'">
    Your MySQL root password doesn't match the one in `db.php`.
    
    **Solution**: Update the password in `db.php` or reset your MySQL root password:
    ```sql
    ALTER USER 'root'@'localhost' IDENTIFIED BY '';
    FLUSH PRIVILEGES;
    ```
  </Accordion>
  
  <Accordion title="Error: Unknown database 'tienda_ropa'">
    The database wasn't created during import.
    
    **Solution**: Manually create it in phpMyAdmin SQL tab:
    ```sql
    CREATE DATABASE tienda_ropa DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
    ```
    
    Then re-import `TiendaRopa.sql`.
  </Accordion>
  
  <Accordion title="Tables are empty after import">
    The INSERT statements may have failed.
    
    **Solution**: Check phpMyAdmin's import log for errors. Ensure:
    1. The SQL file is complete and not corrupted
    2. You have sufficient MySQL privileges
    3. No foreign key constraint errors occurred
  </Accordion>
  
  <Accordion title="Connection error when accessing the application">
    Check these common issues:
    
    1. **MySQL not running**: Start it in XAMPP Control Panel
    2. **Wrong credentials**: Verify `db.php` matches your MySQL setup
    3. **PDO not enabled**: Check PHP extensions in `php.ini`
    
    Test connection manually:
    ```php
    <?php
    try {
        $conn = new PDO("mysql:host=localhost;dbname=tienda_ropa", "root", "");
        echo "Connection successful!";
    } catch(PDOException $e) {
        echo "Error: " . $e->getMessage();
    }
    ?>
    ```
  </Accordion>
  
  <Accordion title="Foreign key constraint errors">
    This happens if you try to insert/update data that violates relationships.
    
    **Solution**: The SQL file creates tables in the correct order. Always:
    1. Drop all tables if re-importing
    2. Import the complete SQL file
    3. Don't manually modify table structure without updating constraints
  </Accordion>
</AccordionGroup>
